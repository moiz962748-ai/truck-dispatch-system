## File Details & Features

### Configuration Files
- **vite.config.js**: Vite build configuration with React plugin
- **tailwind.config.js**: Tailwind CSS setup
- **postcss.config.js**: PostCSS with autoprefixer
- **package.json**: Dependencies (React, Router, Axios, Framer Motion, Tailwind)

### Entry Point
- **index.html**: Root HTML with div#root
- **src/main.jsx**: React app initialization with Router

### Core App
- **App.jsx**: Routes all pages, wraps with Navbar & Footer

### Styling
- **src/index.css**: Global styles, blob animations, smooth scroll

### API Layer
- **src/api/api.js**:
  - Axios instance with baseURL
  - JWT interceptor for all requests
  - Functions: loginUser, registerUser, fetchMyLoads, createLoad, fetchAllLoads, fetchDrivers, assignDriver, updateLoadStatus

### Components
- **Navbar.jsx** (180 lines):
  - Sticky header with logo
  - Desktop navigation menu
  - Mobile hamburger menu with animation
  - Conditional login/logout buttons
  - Responsive design

- **Footer.jsx** (50 lines):
  - Multi-column footer layout
  - Links to product, company, support
  - Social media links
  - Copyright info

- **LoadForm.jsx** (80 lines):
  - Animated input fields with Framer Motion
  - Pickup, dropoff, weight fields
  - Success/error toast messages
  - Loading spinner
  - Form submission handler

- **PrivateRoute.jsx** (20 lines):
  - Checks JWT token existence
  - Checks user role matches allowedRoles
  - Redirects to login if unauthorized
  - Preserves location for post-login redirect

## Animation Features

### Framer Motion Animations
- Component entrance animations: `initial` → `animate`
- Hover effects: `whileHover={{ scale: 1.02 }}`
- Tap effects: `whileTap={{ scale: 0.98 }}`
- Layout animations: `motion.div` components
- Staggered animations with `delay`

### CSS Animations
- **Blob animations** in hero section (7s loop)
- **Smooth scroll** behavior
- **Input focus** transitions
- **Fade-in** on scroll (IntersectionObserver)
- **Status highlights** (green background)
- **Custom scrollbar** styling

## API Integration Points

All API calls use JWT token stored in localStorage:

```
Authorization: Bearer <token>
```

Backend endpoints expected:
- POST /auth/login
- POST /auth/register
- GET /loads (all)
- GET /loads/my (driver's)
- POST /loads
- PUT /loads/:id/assign
- PUT /loads/:id/status
- GET /drivers

## Responsive Breakpoints

- **Mobile**: < 640px (sm)
  - Hamburger menu in Navbar
  - Single column layouts
  - Full-width inputs and buttons
  
- **Tablet**: 640px - 1024px (md)
  - Desktop navigation visible
  - 2-column grids
  - Horizontal form layouts
  
- **Desktop**: > 1024px (lg)
  - Full navigation
  - 3-4 column grids
  - Multi-row tables

## Performance Considerations

- Lazy component loading with React Router
- Efficient atom state with useState hooks
- Optimized re-renders with proper dependencies
- Debounced API calls
- Image/blob optimization with CSS animations
- Minimal JavaScript bundle with tree-shaking
